Конечно, давай структурируем метрики так, чтобы у тебя было четкое разделение: что видит игрок, а что «под капотом» обсчитывает твоя программа.
## 1. То, что ВИДИТ пользователь (Экранная статистика)
Эти параметры должны обновляться в реальном времени (в твоем Tick или через EventBus).

| Метрика | Что это | Формула для кода |
|---|---|---|
| WPM (Words Per Minute) | Скорость в словах | (TotalTyped / 5.0) / (Seconds / 60.0) |
| CPM (Chars Per Minute) | Чистая скорость в знаках | TotalTyped / (Seconds / 60.0) |
| Accuracy (%) | Точность попаданий | (double)(TotalTyped - Errors) / TotalTyped * 100 |
| Time | Таймер (00:00) | Stopwatch.Elapsed (формат mm\:ss) |
| Errors Count | Число ошибок | Просто счетчик Errors |


* Нюанс для WPM: В мире печати принято считать, что 1 слово = 5 символов (включая пробелы). Поэтому мы делим общее число нажатий на 5.

------------------------------
## 2. То, что ты СОБИРАЕШЬ (Скрытые данные)
Эти переменные нужны тебе для расчетов, но выводить их на главный экран не обязательно (они загромождают интерфейс).

* TotalTyped (int): Общее количество нажатых клавиш (кроме Backspace). Это главный "знаменатель" для всех формул.
* Errors (int): Общее число промахов.
* CorrectCount (int): Количество верно нажатых клавиш. Нужен для проверки завершения текста (CorrectCount == FullText.Length).
* StartTime (DateTime): Время самого первого нажатия (чтобы понимать, когда реально началась сессия, а не просто запущен таймер).
* KeyTimestamps (List): Время каждого нажатия.
* Зачем: Чтобы в конце игры построить график ритмичности или найти «затыки» в словах.

------------------------------
## 3. Продвинутые метрики (Для экрана итогов)
Когда игрок закончил текст, покажи ему «глубокую» аналитику:
## Ритмичность (Consistency)
Показывает, насколько равномерно игрок нажимает на клавиши.

* Как считать: Вычисли среднее отклонение времени между нажатиями.
* Зачем: Если игрок печатает рывками (быстро простые слоги и долго ищет сложные буквы), его ритмичность будет низкой (например, 45%). Мастера печатают как пулемет — ровно.

## Проблемные клавиши (Top Errors)

* Как считать: Сделай словарь Dictionary<char, int>, где ключ — буква, в которой ошиблись, а значение — сколько раз.
* Что показать: «Твои самые слабые буквы: Х, Ъ, Э».

## Net WPM (Чистая скорость)
Скорость с учетом «штрафа» за ошибки.

* Формула: ( (TotalTyped - Errors) / 5.0 ) / (Seconds / 60.0)
* Зачем: Это самая честная метрика. Если игрок просто бил по клавишам со скоростью 500 зн/мин, но сделал 50 ошибок, его Net WPM будет очень низким.

В твоей ViewModelStatisticPanel создай метод RecalculateAll(), который будет вызываться в Tick.

private void RecalculateAll()
{
    double minutes = _stopwatch.Elapsed.TotalMinutes;
    if (minutes > 0)
    {
        Velocity = TotalTyped / minutes; // Это CPM
        Wpm = (TotalTyped / 5.0) / minutes; 
    }
    AccuracyCount();
}
